Formatting

These are the rules to abide to when writing in a codebase. It is important to follow established conventions. Formatting has one purpose and that is to make code readable. Formatting is all about consistency to create a maintainable product.

Vertical Alignment

Vertical alignment describes the lines of code written in files. How we use vertical alignment is purely up to the programmer. We vertically look over functions to determine how they work and how they are used as a whole.

We may group code by having lines immediately proceed each other and ungroup them for separating them into different ideas. In this way lines can be easily read at a glance. Similarly, variable declarations are placed vertically above and as close as possible to where they are called.

Variables and functions should also be grouped together by their type and usage in a vertical manner. Having setters and getters distributed in the same vicinity helps provide readability by separating the business logic from the encapsulating methods.

Horizontal Alignment

Horizontally, we read lines to determine what is happening. We like to have short lines and often try to shorten lines by creating functions that do operations elsewhere.

Horizontal indentation also explains the scope of variables. The scope determines what value is being used in an environment by binding a value to a variable.

Python is a unique case where indentation truly matters. In Python, indentation determines the code blocks and the scope of variables. The result is that Python naturally provides readable coding structures.

In C-style languages, indentation is purely stylistic in that it has no effect on the execution. Instead, curly brackets are used to indicate code blocks.
